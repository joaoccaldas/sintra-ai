**DB Schema (SQLite):**
```sql
CREATE TABLE transactions (
  id TEXT PRIMARY KEY,
  date TEXT, amount REAL, merchant TEXT,
  category TEXT, raw_category TEXT, flagged INTEGER DEFAULT 0
);
CREATE TABLE daily_summaries (date TEXT PRIMARY KEY, json TEXT);
```

**Ingestion pipeline:**
1. Pull transactions via Plaid /transactions/sync (or parse CSV)
2. Upsert into SQLite (dedup by transaction id)
3. For each uncategorised row, call Claude with merchant name + amount → returns category

**Categorisation prompt:**
```
Classify this transaction into one of: [Food, Transport, Housing, Subscriptions, Shopping, Health, Entertainment, Other].
Merchant: {merchant}  Amount: ${amount}
Return only the category name.
```

**Anomaly detection logic:**
- Flag if amount > 2× rolling 30-day average for that category
- Flag if merchant appears for the first time and amount > $50

**Daily briefing prompt:**
```
Given these transactions from {date}: {json}
And monthly budget targets: {targets}
Write a 5-bullet morning briefing covering: total spend, top category, biggest single charge, any flagged anomalies, and one actionable suggestion.
