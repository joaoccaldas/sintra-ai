The prompt

Refactor the following legacy code to modern standards without changing its external behavior. Legacy code: [paste the code here] Language: [e.g., Python 2 → 3.12, JavaScript ES5 → ES2022, Java 8 → 21] Known pain points: [e.g., 'hard to test', 'no error handling', 'global mutable state', 'spaghetti logic'] Refactoring goals (check all that apply): [list from: readability, testability, performance, security, modularity] Please: 1. Produce the refactored code with all modernizations applied 2. List every change made with a one-line rationale (format: old pattern → new pattern: reason) 3. Highlight any behavior that changed even slightly and explain the trade-off 4. Identify any remaining tech debt that is out of scope for this refactor 5. Suggest 3 follow-up refactors ordered by ROI 6. Provide a before/after complexity comparison (cyclomatic complexity if possible) Constraint: preserve the existing public API/function signatures unless a signature change is justified.
