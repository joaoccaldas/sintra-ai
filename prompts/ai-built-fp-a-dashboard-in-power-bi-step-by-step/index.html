AI-Built FP&A Dashboard in Power BI — Step by Step

The prompt

"I want to build a Finance / FP&A dashboard in Power BI using AI to accelerate every step. Dashboard purpose: [e.g. Monthly P&L review / Budget vs Actual / Cash flow monitoring] Audience: [e.g. CFO + Finance team] Data sources: [e.g. ERP export in Excel, General Ledger CSV, Headcount table] Key financial metrics to show: [list 5–8, e.g. Revenue, COGS, Gross Margin, OPEX, EBITDA, Headcount] Fiscal year: [Jan–Dec / Feb–Jan / other] For each stage of the build, give me: 1. DATA PREP — the AI prompt to clean and structure the source data in Power Query 2. MODELLING — the DAX measures to create (with code) 3. VISUAL DESIGN — the Copilot prompt to build each chart/KPI 4. COMMENTARY — the AI prompt to auto-write the CFO narrative for this month 5. SHARING — how to set up row-level security so each region only sees their own data"
Open in ChatGPT

Sample output

**1. DATA PREP — Power Query AI prompt:** "Clean this General Ledger export: remove blank rows, split the Account Code column into Department and Account Number, standardise all date formats to YYYY-MM-DD, and flag any negative Revenue lines as 'Adjustment' in a new Type column." **2. KEY DAX MEASURES:** ```dax Gross Margin % = DIVIDE([Gross Profit], [Total Revenue], BLANK()) Budget Variance % = DIVIDE([Total Revenue] - SUM('Budget'[Revenue]), SUM('Budget'[Revenue])) EBITDA = [Gross Profit] - [Total OPEX] + [D&A] ``` **3. COPILOT VISUAL PROMPT:** "Build a P&L summary page with: a KPI row (Revenue, Gross Margin %, EBITDA, Headcount), a waterfall chart showing Revenue → Gross Profit → EBITDA bridge, and a small multiples line chart of Revenue by Department vs prior year. Add a Month slicer." **4. CFO COMMENTARY PROMPT:** "Write a 3-paragraph CFO commentary for [Month] [Year]. Revenue was [X] vs budget [Y] ([Z]%). Gross margin was [A]% vs [B]% prior year. OPEX was [C] vs budget [D]. Tone: direct, no jargon, flag two risks and one positive. Format: bullet summary + 2-sentence paragraph per metric." **5. ROW-LEVEL SECURITY:** In Power BI Desktop → Modelling → Manage Roles → create role 'Region Manager' with filter: `[Region] = USERPRINCIPALNAME()`. Publish and assign users in Power BI Service → Dataset → Security.
