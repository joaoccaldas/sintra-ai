The prompt

I'm building a [game genre] game for [target platform]. Help me design the core game loop. Define: the player's primary action, the immediate feedback, the short-term goal (session), the long-term goal (progression), and the re-engagement hook. Then identify the 3 biggest risks that could make the loop feel unsatisfying.
Open in Claude

Sample output

**Game:** Tower Defense Roguelite (Mobile) **Core loop:** - **Primary action:** Place and upgrade towers using earned currency - **Immediate feedback:** Visual + haptic pop when a tower kills an enemy; currency counter ticks up - **Session goal (5–10 min):** Survive N waves and reach the boss - **Progression goal:** Unlock permanent upgrades, new tower types, harder maps - **Re-engagement hook:** Daily challenge with unique constraint; streak reward system **Top 3 risks:** 1. **Upgrade overwhelm** — Too many options at once kills mobile pacing. Solution: gate by wave milestone. 2. **No comeback mechanic** — If a run goes bad early, players quit. Add a 'Last Stand' emergency ability. 3. **Progression plateau** — If permanent upgrades cap out, endgame feels flat. Plan a prestige system from day one.
