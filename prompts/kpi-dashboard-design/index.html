The prompt

Design a KPI dashboard for the following business function. Be specific enough that a BI developer could build it from your output. Business function: [e.g., Sales, Marketing, Customer Success, Engineering, Finance, Operations] Stakeholder audience: [who will read this dashboard — e.g., VP of Sales, SDR team, C-suite] Decisions this dashboard should drive: [list 2–3 specific decisions the audience makes using this data] Data sources available: [e.g., Salesforce, HubSpot, PostgreSQL, Stripe, Google Analytics] Refresh cadence needed: [real-time / hourly / daily / weekly] Deliver: 1. KPI hierarchy — tier 1 (north-star), tier 2 (drivers), tier 3 (operational diagnostics) 2. For each KPI: metric name, formula, data source, owner, target value, alert threshold 3. Dashboard layout recommendation — number of sections, chart types, and which KPIs go where 4. Drill-down paths — which tier-1 metric can be decomposed into which tier-2/3 metrics 5. Three metrics that look important but are actually vanity metrics for this audience 6. Data freshness and quality risks — which KPIs are most likely to be wrong and why 7. Implementation checklist — 5 steps to go from zero to a live dashboard in a BI tool
