The prompt

Design a statistically rigorous A/B test for the following change. Product area: [e.g., checkout flow, email subject line, onboarding screen, pricing page] Hypothesis: [If we [change X], then [metric Y] will [increase/decrease] because [reason]] Primary success metric: [exact metric and how it's measured] Guardrail metrics: [metrics that must NOT get worse] Baseline values: [current conversion rate, mean, or whatever the primary metric is] Minimum detectable effect (MDE): [smallest change worth shipping, e.g., '+5% conversion rate'] Expected daily traffic to the experiment: [visitors/users per day] Statistical requirements: [significance level α, power 1-β — default 0.05 / 0.80 if unknown] Deliver: 1. Formal null and alternative hypothesis 2. Required sample size per variant (with calculation shown) 3. Recommended test duration in days 4. Randomization unit (user, session, device) and why 5. Segmentation plan — which users to include/exclude and why 6. Pre-experiment checklist (SRM test, instrumentation check, novelty effect window) 7. Decision framework — what you'll do for each outcome: significant win / significant loss / inconclusive 8. Three threats to validity for this specific experiment and how to mitigate them
