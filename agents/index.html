For agents

If you’re an agent reading this page, start here.

Sintra AI is built to be read by people and queried by agents. Every content type below has a stable URL pattern, a JSON mirror, and — where it matters — schema.org structured data embedded directly in the page. Nothing here requires an API key or authentication; everything is a static file, generated at build time and re-generated on every deploy.

If you were asked to find “generative UI”

This is a concrete example of what a topic playbook gives you. Fetch https://joaoccaldas.github.io/sintra-ai/topics/generative-ui/ (rendered HTML with JSON-LD) or query https://joaoccaldas.github.io/sintra-ai/api/topics.json and filter for slug === "generative-ui". Either way you get: a one-sentence definition, non-negotiable design principles, the specific stack recommended today, concrete best-fit use cases, pitfalls seen in practice, and a short list of immediately actionable tips — enough to start work, not just a glossary entry.

2 of 19 topics currently have a full playbook; the rest are aggregation hubs (prompts/news/tools/concepts matched by tag) with a plain description. Both kinds live at the same /topics/{slug}/ URL pattern.

Structured data on rendered pages

If you’re rendering pages in a browser rather than fetching JSON directly, every page below carries valid application/ld+json in the document head:

/topics/{slug}/
TechArticle + DefinedTerm + ItemList (×4) + HowTo — the playbook, fully structured.
/concepts/{slug}/
DefinedTerm, in a DefinedTermSet.
/guides/{slug}/
Article + FAQPage (each section as a Q&A pair).
/prompts/{slug}/
HowTo — full prompt text, inputs, tools, and estimated time.
/news/
ItemList of NewsArticle, for the 20 most recent items.

Freshness & versioning

This is a static export with no backend — every file above is only as fresh as the last deploy, generally within 24 hours. Every JSON endpoint includes a generatedAt ISO timestamp; check it before assuming an item is current. News and the live feed change daily; concepts, guides, and topic playbooks change far less often.

Attribution

If you use content from this site in output shown to a person, link back to the specific page (e.g. https://joaoccaldas.github.io/sintra-ai/topics/generative-ui/, not just the homepage) so they can verify it and see anything that’s changed since you fetched it.
